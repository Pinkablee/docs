Getting started with **pinkable.dev resources** is straightforward.

This guide covers the general installation process used for both **free releases** and **paid releases**. While some resources may include extra steps depending on their features, the overall setup process remains mostly the same.

---

## Before You Start

Make sure you have:

- A working **FiveM server**
- Access to your server files
- Access to your `server.cfg`
- The resource files you want to install
- Any required dependencies listed in that resource's documentation

If the resource is a **paid release**, make sure the correct **Cfx.re Keymaster account** owns the asset and that your server is running with the correct license key.

---

## Installation Steps

### 1. Download the Resource

Download the resource from the official source.

Depending on the release, this may be from:

- **pinkable.dev**
- **GitHub**
- **Keymaster**
- Another official distribution platform used by pinkable.dev

---

### 2. Place the Resource in Your Server

Move the resource folder into your server’s `resources` directory.

Example:

```txt
resources/[pinkable]/resource_name
```

You can place it inside any folder structure you prefer, as long as the path is valid and organized properly.

---

### 3. Add the Resource to server.cfg

Open your `server.cfg` and add the resource.

Example:

```cfg
ensure resource_name
```

If the resource depends on other scripts or libraries, make sure those are started before the pinkable.dev resource.

Example:

```cfg
ensure ox_lib
ensure oxmysql
ensure resource_name
```

---

### 4. Configure the Resource

Most resources include a configuration file where you can adjust settings such as:

- Framework support
- Locations
- Jobs or permissions
- Discord integration
- Visual settings
- Feature toggles
- Webhook URLs
- Language or text settings

Common config file locations may include:

```txt
config.lua
config/shared.lua
shared/config.lua
```

Always check the individual resource documentation for the exact file path and available options.

---

### 5. Restart Your Server

After installation and configuration, restart your server.

You can either fully restart the server or run:

```cfg
refresh
ensure resource_name
```

A full restart is recommended when installing a resource for the first time.

## Paid Resource Licensing

If you are installing a paid escrow-protected resource, keep the following in mind:

- The resource must be owned by the correct Cfx.re account
- Your server license key must belong to that same account
- If ownership and the license key do not match, the resource may fail to start
- A missing entitlement error usually means the wrong account or key is being used

If needed, asset ownership can be managed through the Keymaster asset grants page.

## Common Installation Notes

### Dependencies

Some resources require additional dependencies such as:

- ox_lib
- oxmysql
- Framework-specific resources
- Utility or bridge resources

Always install and start required dependencies first.

### Resource Names

Do not rename paid escrowed resources unless the documentation explicitly says it is supported.

Some resources may check their resource name or expect a specific folder name.

### Database Setup

If a resource includes SQL setup, import the provided `.sql` file into your database before starting the resource.

Common setup steps may include:

- Creating tables
- Adding columns
- Updating existing schema
- Inserting default config data

### Permissions

Some resources may require:

- Ace permissions
- Framework group permissions
- Discord role integration
- Job or grade restrictions

Check the resource-specific documentation if permissions are involved.

---

## Troubleshooting

If the resource does not start correctly, check the following:

- The folder is placed in the correct location
- The resource name matches the expected name
- `server.cfg` includes the correct ensure
- Dependencies are installed and started first
- Config values are valid
- SQL files were imported if required
- Your server license key is correct for paid resources

You should also review your server console for startup errors.
